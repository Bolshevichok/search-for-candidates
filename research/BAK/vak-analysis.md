# Источник: сайт ВАК

Что реально можно получить по кандидату через сайт Высшей аттестационной комиссии, как он устроен технически и как к нему обращаться программно.

**Ключевые цифры:**
- Личный e-mail/телефон кандидата — **нет, не публикуется никогда** (подтверждено на уровне схемы данных, не только вёрстки)
- Возраст / дата рождения — **нет**
- Доступ — **свободный, без капчи, регистрации и защиты от ботов** (проверено вживую, в т.ч. обычным `curl`)
- Всего записей в базе объявлений о защитах — **164 045** (на момент проверки)
- Есть **открытый JSON API** для поиска по ФИО — это основной способ работы с источником

## 1. Домен сменился

Старый адрес `vak.minobrnauki.gov.ru` **больше не основной**. С 19 августа 2025 г. система (ФИС ГНА) переведена на единую цифровую платформу «ГосТех», актуальный адрес:

**https://vak.gisnauka.ru**

## 2. Открытый JSON API — основной способ работы с источником

Через анализ сетевых запросов реального браузера удалось найти, что весь раздел «Объявления о защитах» работает через открытый REST API без авторизации, без API-ключа и без CORS-ограничений:

| Метод | Назначение |
|---|---|
| `GET /api/att/adverts/?page=N&pageSize=M&is_pilot=false` | Список объявлений, постранично |
| `GET /api/att/adverts/?page=1&pageSize=N&search=<ФИО>&is_pilot=false` | **Поиск по ФИО** — проверено, работает, возвращает точное совпадение |
| `GET /api/att/adverts/<uuid>/` | Полная карточка объявления по её id |
| `GET /api/att/adverts/<old_id>/autoref-file/` | Прямая ссылка на PDF автореферата |
| `GET /api/att/branch-list/` | Справочник «Отрасль науки» (для фильтров) |
| `GET /api/att/specialty-list/` | Справочник научных специальностей |
| `GET /api/att/dissertation-type-list/` | Справочник типов диссертации (канд./докт.) |

Проверено вживую: запрос `.../api/att/adverts/?page=1&pageSize=10&search=Зубов Вадим Владиславович&is_pilot=false` вернул `"count":1"` и ровно нужную запись. То есть **для проверки конкретного кандидата по ФИО не нужен никакой HTML-парсинг** — можно обращаться напрямую к этому эндпоинту и получать чистый JSON, а листингом по страницам (`page`/`pageSize`) можно выгрузить всю базу целиком.

Также проверено напрямую через `curl`, без браузера и без каких-либо специальных заголовков — сайт отдаёт `200 OK` и корректный JSON сразу. Единственная особенность: сервер отвечает не мгновенно (порядка 3–5 секунд на запрос) — при выгрузке всей базы через API нужно закладывать таймаут в районе 8–10 секунд на запрос и не удивляться задержке, это нормальный режим работы сайта, а не сбой.

Это подтверждает и закрывает вопрос по полноте данных: **в схеме API физически нет полей под e-mail, телефон или дату рождения кандидата** — значит, надеяться, что они «просто не показывают в вёрстке, но передают в API», не приходится. Только организационные контакты (`org_address`, `org_phone` — это диссовет/вуз, не человек).

### Что видно в карточке — свод

| Поле | Есть? | Комментарий |
|---|---|---|
| ФИО соискателя | да | точное написание, как в дипломе |
| Тема, специальность, отрасль науки | да | шифр научной специальности по номенклатуре |
| Дата защиты (план/факт) | да | косвенная замена «возрасту» — недавно ли получена степень |
| Диссовет + организация места защиты | да | город есть, но это город защиты, не обязательно проживания |
| Ссылки на автореферат / диссертацию / карточку на сайте вуза | да | `autoref_site`, `fulltext_site`, `card_site` — прямые URL |
| Адрес и телефон организации | да | контакты диссовета/вуза, **не кандидата** |
| Личный e-mail / телефон кандидата | нет | физически отсутствует в схеме данных |
| Возраст / дата рождения | нет | физически отсутствует в схеме данных |

## 3. Что это значит для проекта

ВАК — надёжный источник для **точечной проверки конкретного кандидата по ФИО** (через API `search=`), для **массовой выгрузки всей базы** (постраничный листинг `page`/`pageSize`, всего 164 045 записей) и для обогащения карточки кандидата (степень, дата защиты, специальность, вуз защиты, ссылка на автореферат). Он не даёт контактов и не годится для поиска по городу/возрасту напрямую — только по уже известному ФИО или как массив данных для последующего сопоставления с другими источниками.

Технически это выглядит намного проще, чем казалось на первом проходе: вместо HTML-скрейпинга — прямой вызов открытого JSON API, без авторизации и без защиты от ботов. Достаточно обычного HTTP-клиента с таймаутом 8–10 секунд.
