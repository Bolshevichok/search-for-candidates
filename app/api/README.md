# API поиска кандидатов

FastAPI-приложение находится в `app/api/main.py`. Оно читает рабочую SQLite-базу,
запускает полный пайплайн в фоне, позволяет штатно отменить его и отдаёт данные
для будущего интерфейса.

## Запуск

Из корня проекта:

```powershell
uvicorn app.api.main:app --host 127.0.0.1 --port 8000
```

После запуска доступны:

- `http://localhost:8000/docs` — Swagger UI: интерактивная документация и отправка запросов;
- `http://localhost:8000/openapi.json` — OpenAPI-схема в JSON;
- `http://localhost:8000/api/v1/health` — быстрая проверка доступности API и базы.

Swagger создаётся автоматически при вызове `FastAPI(...)` в `create_app()`. Все
маршруты с декораторами `@app.get`, `@app.post` и `@app.delete` попадают в него
без отдельной генерации файла.

## Основные маршруты

| Метод и путь | Назначение |
| --- | --- |
| `GET /api/v1/analytics` | Сводные показатели: вузы, ВАК, кандидаты, контакты, VK и ошибки. |
| `GET /api/v1/universities` | Реестр вузов и домены для фильтров. |
| `GET /api/v1/records/teachers` | Строки листа «Сотрудники вузов» из XLSX. Поддерживает `q`, `domain`, `has_vk`, `limit`, `offset`. |
| `GET /api/v1/records/vak` | Строки листа «Кандидаты ВАК» из XLSX с теми же параметрами. |
| `GET /api/v1/candidates/{candidate_id}` | Детальная запись кандидата, включая все найденные VK-профили. |
| `GET /api/v1/errors` | Ошибки сбора; можно фильтровать по `run_id` и `domain`. |
| `GET /api/v1/runs/options` | Допустимые настройки полного запуска и значения из `config.yaml`. |
| `POST /api/v1/runs` | Запускает полный пайплайн в фоне. |
| `GET /api/v1/runs` | История запусков. |
| `GET /api/v1/runs/{run_id}` | Статус, этап, длительность, ошибка и ссылка на XLSX. |
| `DELETE /api/v1/runs/{run_id}` | Запрашивает штатную отмену активного запуска. |
| `GET /api/v1/runs/{run_id}/xlsx` | Скачивает XLSX успешного запуска. |

## Запуск пайплайна

API принимает только параметры полного пользовательского запуска. Файл
`config.yaml` не меняется: переданные значения действуют только для новой
задачи.

```powershell
Invoke-RestMethod -Method Post `
  -Uri http://localhost:8000/api/v1/runs `
  -ContentType 'application/json' `
  -Body '{
    "domain": "utmn.ru",
    "limits": {
      "max_universities": 1,
      "vak_max_pages": 2,
      "vk_enabled": true,
      "vk_limit": 200,
      "vk_workers": 2
    }
  }'
```

Ответ содержит `run_id`. Его можно передать в `GET /api/v1/runs/{run_id}` для
опроса состояния или в `DELETE /api/v1/runs/{run_id}` для отмены.

Одновременно допускается только один API-запуск. Второй запрос во время работы
первого вернёт `409 Conflict`.

## Отмена

Отмена кооперативная: API ставит задачу в состояние `cancelling`, а сборщики
останавливаются между запросами и заданиями. Уже начатый HTTP- или браузерный
запрос завершается по своему таймауту. Матчинг не разрывается посередине, чтобы
не оставить таблицу кандидатов в частично пересобранном состоянии.

После успешного запуска XLSX сохраняется в `output/candidates_run_<id>.xlsx`.
У отменённого или аварийного запуска готовый XLSX не публикуется.

## Локальный доступ

API предназначен для локальной демонстрации и не имеет авторизации. Запускайте
его на `127.0.0.1` и не публикуйте порт в интернет.
