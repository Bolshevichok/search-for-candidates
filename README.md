# Поиск кандидатов

Консольное приложение собирает сведения о кандидатах

## Что делает

1. Собирает сотрудников с сайтов вузов.
2. Загружает сведения о защитах из ВАК.
3. Сопоставляет людей из двух источников.
4. Создаёт Excel-файл с результатом.
5. Обогощает кандидатов контактами обходя сайты университетов ИИ-парсером и дополняет результат. 


## Установка

Нужен Python 3.12 или новее. Создайте виртуальное окружение проекта, активируйте его и установите зависимости из проекта. Затем скопируйте `.env.example` в `.env`.

Файл `.env` нужен только для второго слоя: если обогощение контактами не требуется, то его можно не запускать.

## Основные команды

```powershell
# Полный основной прогон
python -m app run

# Прогон на пяти вузах для быстрой проверки
python -m app --config config.smoke5.yaml run --out output/smoke5.xlsx

# Проверить состояние рабочей БД
python -m app status

# Очистить рабочую БД
python -m app reset

# Создать XLSX из уже собранных данных
python -m app export --out output/candidates.xlsx
```

## Обогощение записей контактами

Второй слой ищет электронную почту и телефон кандидатов на сайтах вузов. Для него нужны `YANDEX_FOLDER_ID` и `YANDEX_API_KEY` в `.env`.

```powershell
python -m app step layer2 --limit 100
```

## Обогащение профилями VK без API

```powershell
# Для ТюмГУ: один последовательный индекс публичных подписчиков, затем
# обработка максимум 20 кандидатов из базы. Логин и VK API не нужны.
python -m app --domain utmn.ru step vk --limit 20
```


## Структура проекта

### Корень проекта

| Файл или папка | Что в ней и зачем нужна |
| --- | --- |
| `pyproject.toml` | Зависимости проекта |
| `.env.example` | Шаблон для переменных |
| `.gitignore` | Список того чего не стоит коммитить в репо |
| `config.yaml` | Основные лимиты, задержки запросов и число параллельных задач для обычного прогона. |
| `config.smoke5.yaml` | Настройки быстрой проверки на пяти вузах. |
| `config.layer2.yaml` | Пример настроек для обогащения кандидатов контактами. |
| `config.quickstart.yaml` | Минимальный пример конфигурации для прогона на двух вузах. |

### Приложение `app/`

Здесь находится основной код, который запускается для парсинга сайтов университетов, ВАК и ВК.

Файлы `__init__.py` в папках нужны Python, чтобы считать их частями одного приложения.


| Файл или папка | Что в ней и зачем нужна |
| --- | --- |
| `__main__.py` | Точка входа для команды `python -m app`. |
| `cli.py` | Описывает команды `run`, `step`, `status`, `reset` и `export`, запускает нужные этапы. |
| `config.py` | Читает YAML-конфигурацию и переменные из `.env`. |
| `db/schema.sql` | Структура рабочей SQLite-базы |
| `db/repository.py` | Единое место для чтения и записи данных в SQLite. |
| `pipeline/ingest.py` | Координирует основной сбор: первый слой и ВАК. |
| `registry/loader.py` | Загружает реестр вузов из CSV в рабочую базу. |
| `sources/http_client.py` | Общий HTTP-клиент с повторными попытками запросов и обработкой ошибок. |
| `sources/universities/layer1.py` | Первый слой: обходит сайты вузов и извлекает сотрудников. |
| `sources/universities/struct.py` | Помогает первому слою распознавать структуру страниц и кафедры. |
| `sources/universities/layer2.py` | Второй слой: ищет на сайтах вузов электронную почту и телефон уже найденных кандидатов. |
| `sources/vak/client.py` | Клиент для запросов к источнику данных ВАК. |
| `sources/vak/parser.py` | Разбирает ответы ВАК в данные о защитах. |
| `sources/vak/pipeline.py` | Организует массовую загрузку и сохранение данных ВАК. |
| `matching/normalize.py` | Приводит ФИО и названия организаций к единому виду для сравнения. |
| `matching/identity_key.py` | Создаёт ключи и правила сравнения сотрудников. |
| `matching/employee_merge.py` | Убирает повторяющиеся записи сотрудников, собранные с сайта одного вуза. |
| `matching/matcher.py` | Сопоставляет сотрудников вузов со сведениями о защитах ВАК. |
| `export/xlsx.py` | Формирует итоговую XLSX-выгрузку с русскими названиями столбцов, фильтрами и форматированием. |

### Данные `data/` и результаты `output/`

| Файл или папка | Что в ней и зачем нужна |
| --- | --- |
| `data/university_registry.csv` | Исходный реестр вузов. Чтобы вуз обрабатывался, у него должен быть указан домен сайта. |
| `data/state.sqlite` | Рабочая SQLite-база результатов. Её можно безопасно очистить командой `python -m app reset` и собрать заново. |
| `data/raw/` | Крупные исходные файлы, из которых при необходимости собирают реестр вузов. |
| `output/` | Готовые XLSX-файлы для пользователя. |

### Сборка реестра вузов `university_registry/`

Эти скрипты не участвуют в обычном поиске. Они нужны, когда требуется обновить сам файл `data/university_registry.csv`.

| Файл | Что делает |
| --- | --- |
| `download_accred_data.py` | Скачивает исходные данные реестра аккредитаций. |
| `build_university_registry.py` | Собирает из исходных данных итоговый CSV-реестр вузов. |
| `resolve_vk_browser.py` | Ищет группы вузов во «ВКонтакте» через браузер. |

## Локальный API

API запускает полный сбор в фоне, показывает аналитику и отдаёт две выборки с
теми же полями, что и листы XLSX: преподаватели и кандидаты ВАК. Он также
позволяет ограничить прогон доменом, лимитами и числом воркеров, запросить
штатную остановку и скачать XLSX успешного запуска.

Для запуска через Docker Desktop:

```powershell
docker compose up --build
```

Документация API доступна по адресу `http://localhost:8000/docs`. База
`data/state.sqlite` и файлы из `output/` примонтированы в контейнер, поэтому
не пропадают после его остановки.

Для локального запуска без Docker установите зависимости проекта и выполните:

```powershell
uvicorn app.api.main:app --host 127.0.0.1 --port 8000
```

HTTP API предназначен для локальной демонстрации и не содержит авторизации:
не публикуйте порт сервиса в интернет.
